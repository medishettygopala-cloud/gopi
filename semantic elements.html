Understanding Semantic HTML

Posted October 11, 2025 • Beginner

Why semantic tags matter

Semantic elements give meaning to web content for browsers, search engines, and assistive technologies. They improve accessibility and make your markup easier to read and maintain.

Common semantic elements (examples)

  • <header> — page or section header
  • <nav> — navigation links
  • <main> — central content of the document
  • <article> — self-contained composition (blog post, news)
  • <section> — thematic grouping inside content
  • <aside> — tangential content like sidebars
  • <footer> — footer for a page or section
  • <figure> & <figcaption> — images with captions
  • <address> — contact info

Extras: interactive and accessibility-boosting elements

Use <details> and <summary> for progressive disclosure. Use <time> with datetime for machine-readable dates.

Show accessibility tips
  • Always include descriptive alt text on images.
  • Use meaningful link text (avoid "click here").
  • Supply ARIA roles/labels when semantics are insufficient.
